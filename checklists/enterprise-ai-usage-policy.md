# Enterprise AI Usage Policy

This is a practical policy template for internal AI tool usage. Adapt it to your organisation.

## Purpose

This policy defines how employees, contractors, and approved third parties may use AI tools safely and responsibly.

## Scope

This policy applies to:

- public AI chat tools
- AI coding assistants
- AI meeting assistants
- AI search tools
- AI-enabled SaaS platforms
- internal LLM applications
- AI agents and automation tools

## Approved Use

Employees may use approved AI tools for:

- drafting non-confidential text
- summarising approved internal material
- learning and research
- generating code examples for review
- creating test data that does not contain real personal or customer data
- improving documentation and operational runbooks

## Prohibited Use

Employees must not enter the following into unapproved AI tools:

- passwords, API keys, tokens, or secrets
- customer confidential data
- personal data unless explicitly approved
- proprietary source code unless explicitly approved
- unreleased financial or business strategy information
- legal, HR, or disciplinary material
- regulated client data
- security vulnerabilities not already approved for disclosure

## AI-Generated Code

AI-generated code must be treated as untrusted until reviewed. It must pass normal engineering controls including:

- peer review
- security review where relevant
- automated tests
- dependency scanning
- static analysis
- secrets scanning

## AI-Generated Decisions

AI tools must not make final decisions for:

- hiring
- firing
- promotions
- compensation
- credit or financial eligibility
- legal conclusions
- medical advice
- customer-impacting decisions
- security access approval

AI may assist, but accountable humans must review and approve.

## Vendor Approval

New AI tools require approval from security, privacy, and the relevant business owner before use with company data.

## Reporting Issues

Employees must report:

- accidental data entry into unapproved tools
- suspected AI data leakage
- unsafe AI output
- unauthorised AI tool usage
- AI vendor security concerns

## Review

This policy should be reviewed at least every six months or after a material AI incident, vendor change, or regulatory change.
